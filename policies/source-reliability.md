# Source Reliability Policy

## Purpose

Prevent research reports from treating search snippets, marketing articles, social rumors, or model-generated text as verified facts.

## Grades

| Grade | Source | Decision use |
|---|---|---|
| S | Official docs, official announcements, laws/platform rules, code releases, standards | Primary evidence |
| A | Company sites, maintainer blogs, authoritative reports, first-party interviews, formal filings | Strong reference |
| B | Mainstream media, credible blogs/community threads, technical articles | Reference |
| C | Social samples, comments, short videos, forum signals, contextual screenshots | Signal only |
| D | Marketing reposts, unsourced screenshots, one-off rumors, second-hand claims | Lead only |
| unknown | Not graded yet | Do not rely on it |

## Rules

1. Reports should keep source links or source notes.
2. Important claims should include a source grade.
3. S/A sources can support decisions.
4. C/D sources can trigger follow-up validation, but should not be written as market facts.
5. Social signals should be phrased as samples, signals, or hypotheses.
6. When sources conflict, prefer official and first-party evidence.
7. Current-fact reports should include access dates or material dates.
